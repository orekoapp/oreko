# Customer Perspective Analysis: Oreko vs Bloom

*How potential customers perceive and evaluate both products*

---

## Overview

This document analyzes how different customer segments would perceive Oreko and Bloom when evaluating quote/invoice software. Understanding these perspectives helps identify marketing opportunities and product improvements.

---

## Customer Segment Analysis

### Segment 1: Creative Freelancer (Photographer, Designer)

**Profile:**
- Solo practitioner or small studio
- Values aesthetics and professionalism
- Non-technical, prefers "it just works" solutions
- Budget-conscious but willing to pay for value
- Needs client galleries, scheduling, contracts

#### How They Perceive Bloom:

**First Impression:** "This looks like it was made for people like me."
- Sees testimonials from photographers (Peter Hurley, Joel Grimes)
- Recognizes brand names (Dribbble, Girlboss)
- Attracted by "creative professional" messaging
- Gallery/image delivery feature is immediately relevant

**Evaluation:**
- Impressed by comprehensive feature set
- Comfortable with $17/mo for "Solo Business" tier
- Likes mobile app availability
- May feel slightly overwhelmed by options

**Concerns:**
- "Do I really need all these features?"
- "$17/month adds up over a year"
- "What happens to my data if I cancel?"

**Likely Decision:** Signs up for trial, converts if workflow fits

---

#### How They Perceive Oreko:

**First Impression:** "This looks professional but... is it for developers?"
- Sees GitHub link prominently
- "Open Source" badge may confuse non-technical users
- Docker commands in "Open Source" section are intimidating
- Clean design is appealing but less "creative"

**Evaluation:**
- Likes the visual quote builder concept
- Attracted by lower pricing ($9/mo)
- Concerned about lack of galleries
- Uncertain about "self-hosting" (what does that mean?)

**Concerns:**
- "Where are the gallery features?"
- "No mobile app?"
- "What is Docker?"
- "Less testimonials makes me nervous"

**Likely Decision:** May try demo, but likely chooses Bloom for comprehensive features

---

### Segment 2: Tech-Savvy Freelancer (Developer, Consultant)

**Profile:**
- Comfortable with technical tools
- Values data ownership and control
- Skeptical of SaaS subscriptions
- Prefers open-source when possible
- Needs invoicing, not full CRM

#### How They Perceive Bloom:

**First Impression:** "Another SaaS wanting my monthly subscription"
- Skeptical of influencer testimonials
- Notices there's no self-hosting option
- Concerned about vendor lock-in
- Features seem excessive for invoicing needs

**Evaluation:**
- Recognizes it's a polished product
- $17/mo feels expensive for just invoicing
- No API access on lower tiers concerns them
- "What happens to my invoice history if they shut down?"

**Concerns:**
- "I can't inspect the code"
- "My data is on their servers"
- "Too many features I won't use"
- "Another monthly subscription..."

**Likely Decision:** Passes, looks for alternatives or builds own solution

---

#### How They Perceive Oreko:

**First Impression:** "Finally, an open-source option!"
- GitHub link immediately clicked
- "MIT License" triggers positive response
- Docker deployment is familiar and appealing
- Clean, no-nonsense design resonates

**Evaluation:**
- Loves self-hosting capability
- Appreciates focused feature set
- Free tier for self-hosted is compelling
- API access for automation
- Can inspect and modify code if needed

**Concerns:**
- "Is it production-ready?"
- "How active is the community?"
- "Will it be maintained long-term?"

**Likely Decision:** Stars GitHub repo, tries self-hosted version, becomes advocate

---

### Segment 3: Small Agency Owner

**Profile:**
- Manages team of 3-10 people
- Needs collaboration features
- Values efficiency and scalability
- Budget available for tools that work
- Needs invoicing, contracts, project tracking

#### How They Perceive Bloom:

**First Impression:** "This could handle our whole operation"
- "Plus" tier for small teams is appealing
- Workflow management catches attention
- Team features with permissions
- Client portal for professional image

**Evaluation:**
- $33/mo for team features seems reasonable
- Comprehensive but might be feature overkill
- Likes QuickBooks integration
- Mobile apps for team members

**Concerns:**
- "Do we need this much software?"
- "Will team actually adopt all features?"
- "What's the learning curve?"

**Likely Decision:** Likely trial and adopt if team workflow fits

---

#### How They Perceive Oreko:

**First Impression:** "Looks clean but is it enterprise-ready?"
- Limited team features concern them
- Self-hosting could be appealing for data control
- Lower cost is attractive for multiple seats
- Worried about scalability

**Evaluation:**
- Cloud Pro at $19/mo is attractive
- "Up to 3 team members" may be limiting
- No workflow management is a gap
- Would need additional tools for full operation

**Concerns:**
- "Can this scale with us?"
- "Missing project management features"
- "Who handles support issues?"

**Likely Decision:** May choose for invoicing only, supplemented with other tools, OR pass for more comprehensive solution

---

### Segment 4: Cost-Conscious Freelancer (Just Starting Out)

**Profile:**
- New to freelancing or side-hustle stage
- Every dollar matters
- Needs basic invoicing
- Technical comfort varies
- Looking for cheapest viable solution

#### How They Perceive Bloom:

**First Impression:** "$17/month? That's $200/year!"
- Immediately looks at pricing
- Starter at $7/mo (on sale) is more palatable
- Still feels expensive for someone just starting
- Free trial is nice but limited

**Evaluation:**
- Features are impressive but overwhelming
- Worried about 1% transaction fee on Starter
- "Do I really need all this?"
- Compares to free alternatives (Wave, PayPal invoicing)

**Concerns:**
- "I'm not sure I can justify this cost"
- "What if freelancing doesn't work out?"
- "Transaction fees eat into my margins"

**Likely Decision:** Tries free trial, often churns after due to cost

---

#### How They Perceive Oreko:

**First Impression:** "Wait... it's actually FREE?"
- "Open Source" and "FREE forever" catches eye
- Self-hosted option is confusing but intriguing
- $9/mo Cloud Starter if they don't want to self-host
- No transaction fees is major plus

**Evaluation:**
- Free self-hosted is incredibly attractive
- Even Cloud Starter is cheaper than Bloom
- Feature set matches their current needs
- "I can upgrade later if I grow"

**Concerns:**
- "Is free actually free? What's the catch?"
- "Can I trust a free/new product?"
- "What if it disappears?"

**Likely Decision:** High likelihood of trying, especially if somewhat technical

## Key Perception Gaps for Oreko

### 1. "Is This for Developers Only?"
**Problem:** Open source positioning may alienate non-technical users
**Solution:**
- Lead with Cloud option on landing page
- Make Docker/self-hosting secondary
- Add "No technical skills required" messaging
- Show non-developer testimonials

### 2. "Is It Professional Enough?"
**Problem:** Fewer testimonials and no brand recognition
**Solution:**
- Add more social proof (GitHub stars, user count)
- Collect diverse testimonials
- Show real invoice examples
- Add "used by X businesses" counter

### 3. "Does It Have What I Need?"
**Problem:** Focused feature set may seem limited
**Solution:**
- Clear comparison with competitors
- "Everything you need, nothing you don't" messaging
- Roadmap visibility for upcoming features
- Integration ecosystem

### 4. "Will It Be Around?"
**Problem:** New product with uncertain longevity
**Solution:**
- Open source means community can maintain
- Emphasize MIT license (can fork)
- Show GitHub activity/contributors
- Company transparency

---

## Perception Advantages for Oreko

### 1. "Finally, Something That's Actually Free"
- Open source model is genuinely differentiated
- Self-hosting removes ongoing cost
- Even cloud pricing undercuts competitors

### 2. "I Own My Data"
- Privacy-conscious customers value this
- GDPR compliance messaging resonates
- Self-hosting is unique selling point

### 3. "It Does What I Need Without Bloat"
- Customers tired of complex software appreciate focus
- "One thing done well" appeals to minimalists
- Faster to learn and adopt

### 4. "I Can Customize It"
- Developers can extend functionality
- White-label for agencies
- API-first approach

## Customer Journey Considerations

### Bloom Customer Journey:
1. Discovers via influencer recommendation or Google search
2. Impressed by testimonials and comprehensive features
3. Signs up for 7-day trial
4. Explores features, may get overwhelmed
5. Chooses based on specific feature need (galleries, scheduling)
6. Commits to monthly subscription

### Oreko Customer Journey:
1. Discovers via GitHub, Hacker News, or Google search
2. Intrigued by open-source and free pricing
3. Technical users: Clone and self-host
4. Non-technical: Try demo or sign up for cloud
5. Quick evaluation of core invoicing features
6. Adopts if invoicing needs are met

---

## Recommendations for Positioning

### For Technical Audience (Keep):
- GitHub prominence
- Self-hosting instructions
- API documentation
- Open source community

### For Non-Technical Audience (Add):
- "No coding required" messaging
- Cloud-first landing page variant
- Video walkthrough
- More hand-holding in onboarding
- Live chat support option

### For Both Audiences:
- Clearer comparison with competitors
- More testimonials (diverse industries)
- Case studies with real numbers
- "Pricing that makes sense" emphasis

---

## Conclusion

**Bloom** wins perception with non-technical creative professionals due to comprehensive features, influencer trust, and polished brand identity.

**Oreko** wins perception with cost-conscious users, technical freelancers, and privacy-focused customers due to open-source model, focused features, and data ownership.

**The opportunity for Oreko:** Bridge the perception gap with non-technical users without losing the core appeal to technical users. This requires:
1. Dual positioning (technical vs. non-technical landing pages)
2. Stronger social proof
3. Clearer "it just works" messaging for cloud version
4. Maintaining open-source values while being accessible
